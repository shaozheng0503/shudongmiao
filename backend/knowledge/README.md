# 知识库说明（可自行扩充）

Markdown 文件放在本目录任意子文件夹中（例如 `health/`、`emotion/`、`care/`）。**保存文件后无需重启后端**，检索器会在下一次请求时自动重新加载。

## 不会纳入检索的文件

- 文件名以 `_` 开头（例如 `_draft.md`），可用于草稿。
- 根目录或子目录下的 `README.md`（本说明文件不会被当作知识条目）。

## 可选 Frontmatter（YAML 风格）

在正文前使用 `---` 包裹元数据，可提升召回与排序：

```markdown
---
title: 简短标题
tags: 关键词1, 关键词2
scene_hints: stress, general
keywords: 同义词 补充检索词
weight: 1.2
---

正文支持普通 Markdown，写入事实性、可观察的线索与照护建议（非诊断结论）。
```

| 字段 | 说明 |
|------|------|
| `title` | 覆盖默认标题（默认用文件名）。 |
| `tags` | 英文逗号或中文逗号分隔，会参与标签与检索。 |
| `scene_hints` | 与 App 场景对齐：`stress`、`appetite`、`litterbox`、`general`，可多选。 |
| `keywords` / `alias` | 额外检索词（空格或逗号分隔均可），正文不必重复堆砌。 |
| `weight` | 0.1～5.0，略提高该条在相关查询下的排序，默认 1.0。 |

不写 frontmatter 时，行为与此前一致：整篇文件即为正文，标题来自文件名。

## 撰写建议

- 短段落、可扫描；避免与模型提示词中的「非医疗诊断」Disclaimer 冲突。
- 高风险线索（如排尿困难、精神极差）可与现有 `health/` 条目风格保持一致，便于规则与检索协同。
